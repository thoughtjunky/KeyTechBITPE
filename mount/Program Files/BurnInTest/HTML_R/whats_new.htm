What new - Version history

    

V1.3.1004 18/September/2013

       Windows 8.1 support added. Rebooter could display log entries on the Windows Tiled screen after a reboot, this has been corrected.

       Rebooter is now digitally signed.

V1.3.1003 19/June/2009

       Fixed a bug where the "-reboot" command line option would not set the auto start on system startup flag, and hence would only reboot once (regardless on the current configuration).

       Windows 7 support added.

V1.3.1002 29/October/2007

       Fixed a bug in setting auto-login when running under 64-bit versions of Windows.

V1.3.1001 20/March/2007

       Added a command line parameter -p to force Rebooter to use the rebooter.exe directory rather than the User's personal directory for configuration and log files etc. This may be useful when running rebooter from a USB drive. Note: If BurnInTest is started with the -p  command line parameter then BurnInTest will start rebooter with the -p command line parameter.

V1.3.1000 20/February/2007

       Windows Vista release. The help file has been replaced with Browser based help. The configuration and log files have been moved to the User's personal directory on Windows 2000 and above (unless rebooter is run from a BurnInTest installation with a valid BurnInTest key.dat file, e.g. from a CD or flash drive).

V1.2.1001 12/April/2006

       Corrected a bug where a change to the “Auto Load Rebooter at startup” setting was only enabled when Rebooter was started (but not when the user actually changed the setting). The disabling of this setting, as per the user configuration, has been added.

       Corrected a bug where an obscure log file directory name would cause Rebooter to exit.

       Command line parameters may now be specified in the Application field.

V1.2.1000 31/March/2006

       Allow an application to be run between reboots. The application specified in the “application” field on the main window will be run after the timer has expired (reached 0). Rebooter will wait for the external application to close before continuing. If no application is specified, then no application will be run.

       If the log file directory does not exist, it will now be created.

V1.1.1003 1/February/2006

       Change start and stop buttons to reset the reboot counter

V1.0.1003 7/March/2005

       Added a logging window and log file options

       Added pause button

       Added stop cycle button

       Added option to auto load Rebooter at startup

V1.0.1001 13/April/2002

       Changed name of the Rebooter configuration file to avoid a conflict with BurnInTest.

V1.0.1000 14/Mar/2002

       First release
